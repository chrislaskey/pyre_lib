# Shipping Best Practices

No automated shipping was performed for this feature. Use the following guidelines when manually creating the branch and PR.

## Branch Naming

- Use `feature/` prefix for new features, `fix/` for bug fixes
- Keep names short, lowercase, kebab-case
- Example: `feature/products-listing-page`

## Commit Messages

- Use conventional commit format: `feat:`, `fix:`, `refactor:`
- First line under 72 characters
- Include a body summarizing what changed and why

## Pull Requests

- PR title should be concise (under 70 chars) and describe the change
- PR body should include a summary, list of changes, and testing notes
- Reference the feature requirements in the description
